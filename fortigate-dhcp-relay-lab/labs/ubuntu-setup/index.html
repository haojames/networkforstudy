🐧 Cấu hình Ubuntu Server ra Internet

Hướng dẫn chi tiết để Ubuntu có thể ping ra Internet và cài đặt package

🎯 Mục tiêu

Cấu hình mạng cho Ubuntu Server để có thể ping ra Internet và cài đặt các gói cần thiết.

🔍 Kiểm tra trạng thái hiện tại

ip a

Card mạng (thường là ens3 hoặc eth0) ở trạng thái UP nhưng chưa có IP.

🔧 Cấu hình IP tĩnh với Netplan

Bước 1: Xác định file cấu hình

ls /etc/netplan/

Bước 2: Chỉnh sửa file cấu hình

sudo nano /etc/netplan/01-netcfg.yaml

Bước 3: Thêm nội dung

network:
  version: 2
  renderer: networkd
  ethernets:
    ens3:                      # Thay bằng tên card mạng của bạn
      dhcp4: no
      addresses:
        - 172.16.10.10/24
      gateway4: 172.16.10.2
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Bước 4: Áp dụng cấu hình

sudo netplan apply

Bước 5: Kiểm tra lại IP

ip a show ens3
✅ Thành công! Ubuntu đã có IP 172.16.10.10/24

🌐 Cấu hình Firewall Policy trên FortiGate

Để Ubuntu ping ra Internet, cần policy NAT trên FortiGate:

config firewall policy
    edit 1
        set name "Ubuntu-to-Internet"
        set srcintf "port3"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set nat enable
    next
end
📌 Giải thích: set nat enable cho phép Ubuntu sử dụng địa chỉ IP WAN của FortiGate để ra Internet.

✅ Kiểm tra kết nối

1. Ping đến gateway (FortiGate)

ping 172.16.10.2

✅ Kết quả: 64 bytes from 172.16.10.2: icmp_seq=1 ttl=64 time=0.8ms

2. Ping ra Internet

ping 8.8.8.8

✅ Kết quả: 64 bytes from 8.8.8.8: icmp_seq=1 ttl=55 time=15.2ms

3. Ping có tên miền

ping google.com

✅ Kết quả: Phân giải được tên miền và ping thành công

4. Cập nhật danh sách package

sudo apt update

✅ Kết quả: Hiển thị danh sách repository đã cập nhật, không có lỗi

🎉 Thành công! Ubuntu đã có thể truy cập Internet để cài đặt package.

📦 Cài đặt DHCP Server

sudo apt install isc-dhcp-server -y

🐞 Troubleshooting

LỗiCách khắc phục
ping: connect: Network is unreachableKiểm tra lại gateway4 trong file Netplan
ping: 8.8.8.8: Temporary failureThêm default route: sudo ip route add default via 172.16.10.2
apt update báo lỗi DNSKiểm tra nameservers trong file Netplan
