🔄 DHCP Relay trên FortiGate

Hướng dẫn cấu hình FortiGate làm DHCP Relay Agent | Lab trên EVE-NG

📌 Giới thiệu

DHCP Relay cho phép FortiGate chuyển tiếp gói tin DHCP từ Client ở subnet này đến DHCP Server ở subnet khác.

🎯 Mục tiêu lab:
  • ✅ Cấu hình FortiGate làm DHCP Relay Agent
  • ✅ Cấu hình Ubuntu DHCP Server
  • ✅ Client Windows nhận IP từ DHCP Server ở subnet khác

🐧 Phần 1: Cấu hình Ubuntu ra Internet

Bước 1: Cấu hình IP tĩnh

sudo nano /etc/netplan/01-netcfg.yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens3:
      dhcp4: no
      addresses:
        - 172.16.10.10/24
      gateway4: 172.16.10.2
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
sudo netplan apply

Bước 2: Cấu hình Policy NAT trên FortiGate

config firewall policy
    edit 1
        set name "Ubuntu-to-Internet"
        set srcintf "port3"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set nat enable
    next
end

Bước 3: Kiểm tra kết nối

ping 8.8.8.8
ping google.com
✅ Thành công! Ubuntu đã có Internet.

📦 Phần 2: Cài đặt DHCP Server trên Ubuntu

Bước 1: Cài đặt

sudo apt update
sudo apt install isc-dhcp-server -y

Bước 2: Cấu hình dhcpd.conf

sudo nano /etc/dhcp/dhcpd.conf
subnet 10.10.10.0 netmask 255.255.255.0 {
    range 10.10.10.100 10.10.10.200;
    option routers 10.10.10.1;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    default-lease-time 600;
    max-lease-time 7200;
}

subnet 172.16.10.0 netmask 255.255.255.0 {
}

Bước 3: Cấu hình interface

sudo nano /etc/default/isc-dhcp-server
INTERFACESv4="ens3"

Bước 4: Khởi động service

sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
sudo systemctl status isc-dhcp-server

🔄 Phần 3: Cấu hình DHCP Relay trên FortiGate

Bước 1: Cấu hình IP các interface

config system interface
    edit "port2"
        set mode static
        set ip 10.10.10.1 255.255.255.0
        set allowaccess ping
    next
    edit "port3"
        set mode static
        set ip 172.16.10.2 255.255.255.0
        set allowaccess ping
    next
end

Bước 2: Bật DHCP Relay trên port2

config system interface
    edit "port2"
        set dhcp-relay-service enable
        set dhcp-relay-ip "172.16.10.10"
    next
end
⚠️ Quan trọng: DHCP Server (172.16.10.10) và Client (10.10.10.x) phải ở KHÁC subnet!

Bước 3: Tạo Local-In Policy (FortiOS 6.4 - CLI)

config firewall local-in-policy
    edit 1
        set intf "port2"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set service "DHCP"
        set schedule "always"
    next
end
💡 Mẹo: Để xem Local-In Policy trên GUI v6.4: System → Feature Visibility → Bật Local-In Policy, sau đó vào Policy & Objects → Local In Policy.

Bước 4: Tạo Firewall Policy cho DHCP Relay

config firewall policy
    edit 2
        set name "DHCP-Server-to-Client"
        set srcintf "port3"
        set dstintf "port2"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

✅ Phần 4: Kiểm tra hoạt động

Kiểm tra 1: Windows Client nhận IP

ipconfig /renew
ipconfig /all

✅ Kết quả mong đợi: Nhận IP trong dải 10.10.10.100 - 10.10.10.200

Kiểm tra 2: Xem DHCP lease trên FortiGate

execute dhcp lease-list

Kiểm tra 3: Debug DHCP Relay

diagnose debug application dhcprelay -1
diagnose debug enable
# Tắt debug:
diagnose debug disable

Kiểm tra 4: Bắt gói tin DHCP

diagnose sniffer packet port2 "port 67 or port 68" 4
🎉 Hoàn thành! Nếu Windows nhận được IP 10.10.10.xxx, lab DHCP Relay đã thành công!
