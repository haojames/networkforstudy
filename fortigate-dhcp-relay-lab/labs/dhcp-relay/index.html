🔄 DHCP Relay on FortiGate

Complete guide to configure FortiGate as DHCP Relay Agent | Lab on EVE-NG

📌 Introduction

DHCP Relay allows FortiGate to forward DHCP packets from clients in one subnet to a DHCP server located in a different subnet.

🎯 Lab Objectives:
  • ✅ Configure FortiGate as DHCP Relay Agent
  • ✅ Setup Ubuntu DHCP Server
  • ✅ Windows Client receives IP from DHCP Server in different subnet

🐧 Part 1: Configure Ubuntu to Access Internet

Step 1: Configure Static IP with Netplan

sudo nano /etc/netplan/01-netcfg.yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens3:
      dhcp4: no
      addresses:
        - 172.16.10.10/24
      gateway4: 172.16.10.2
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
sudo netplan apply

Step 2: Configure NAT Policy on FortiGate

config firewall policy
    edit 1
        set name "Ubuntu-to-Internet"
        set srcintf "port3"
        set dstintf "wan1"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
        set nat enable
    next
end

Step 3: Verify Connectivity

ping 8.8.8.8
ping google.com
✅ Success! Ubuntu now has Internet access.

📦 Part 2: Install and Configure DHCP Server on Ubuntu

Step 1: Install isc-dhcp-server

sudo apt update
sudo apt install isc-dhcp-server -y

Step 2: Configure dhcpd.conf

sudo nano /etc/dhcp/dhcpd.conf
# Assign IPs to clients in subnet 10.10.10.0/24
subnet 10.10.10.0 netmask 255.255.255.0 {
    range 10.10.10.100 10.10.10.200;
    option routers 10.10.10.1;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    default-lease-time 600;
    max-lease-time 7200;
}

# Declare Ubuntu's own subnet (no IP assignment needed)
subnet 172.16.10.0 netmask 255.255.255.0 {
}

Step 3: Configure Listening Interface

sudo nano /etc/default/isc-dhcp-server
INTERFACESv4="ens3"

Step 4: Start DHCP Service

sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
sudo systemctl status isc-dhcp-server

🔄 Part 3: Configure DHCP Relay on FortiGate

Step 1: Configure Interface IPs

config system interface
    edit "port2"
        set mode static
        set ip 10.10.10.1 255.255.255.0
        set allowaccess ping
    next
    edit "port3"
        set mode static
        set ip 172.16.10.2 255.255.255.0
        set allowaccess ping
    next
end

Step 2: Enable DHCP Relay on port2

config system interface
    edit "port2"
        set dhcp-relay-service enable
        set dhcp-relay-ip "172.16.10.10"
    next
end
⚠️ Important: DHCP Server (172.16.10.10) and Client (10.10.10.x) must be on DIFFERENT subnets!

Step 3: Create Local-In Policy (FortiOS 6.4 - CLI only)

config firewall local-in-policy
    edit 1
        set intf "port2"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set service "DHCP"
        set schedule "always"
    next
end
💡 Tip: To view Local-In Policy on GUI v6.4: System → Feature Visibility → Enable Local-In Policy, then go to Policy & Objects → Local In Policy.

Step 4: Create Firewall Policy for DHCP Relay

config firewall policy
    edit 2
        set name "DHCP-Server-to-Client"
        set srcintf "port3"
        set dstintf "port2"
        set srcaddr "all"
        set dstaddr "all"
        set action accept
        set schedule "always"
        set service "ALL"
    next
end

✅ Part 4: Verify Operation

Test 1: Windows Client receives IP

ipconfig /renew
ipconfig /all

✅ Expected result: IP in range 10.10.10.100 - 10.10.10.200

Test 2: View DHCP lease on FortiGate

execute dhcp lease-list

Test 3: Debug DHCP Relay

diagnose debug application dhcprelay -1
diagnose debug enable
# Disable debug:
diagnose debug disable

Test 4: Capture DHCP packets

diagnose sniffer packet port2 "port 67 or port 68" 4

Test 5: Final Result - Windows Client Receives IP

After running ipconfig /renew, the Windows client successfully receives an IP address from the DHCP server via FortiGate Relay:
